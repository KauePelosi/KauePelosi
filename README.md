Sou um desenvolvedor com foco em **desenvolvimento backend**.  
Me importo em entender como as coisas funcionam por baixo das abstrações — memória, performance e os mecanismos que a maioria prefere ignorar.<br/>
Atualmente construindo um portfólio técnico e avançando continuamente como desenvolvedor.


---

### Áreas de foco

**Baixo nível & Sistemas** — Alocadores de memória, aritmética de ponteiros, benchmarks de performance. Já implementei um alocador linear do zero e comparei com alocação na stack e `malloc/free`. É aqui que minha curiosidade mora.

**Algoritmos & Estruturas de Dados** — Prática ativa de LeetCode em C++. Foco em desenvolver intuição genuína para resolução de problemas, não só decorar padrões.

**Backend & Web** — Aprendendo TypeScript e Angular para completar o stack. Prefiro entender o que um framework faz antes de deixá-lo fazer as coisas por mim.

### Projetos

**[mini-alloc](https://github.com/KauePelosi/mini-alloc)** &nbsp;`C++`  
Alocador de memória customizado implementado do zero. Inclui benchmarks comparando alocação na stack, `malloc/free` e o alocador linear. Feito pra entender gerenciamento de memória no nível de implementação — não só na teoria.

**[mini-shell](https://github.com/KauePelosi/mini-shell)** &nbsp;`C++`  
Um shell Unix mínimo. Faz parsing e execução de comandos, suporta pipes simples. O tipo de projeto que faz você apreciar o trabalho que o `bash` faz.

**[leetcode-cpp](https://github.com/KauePelosi/leetcode-cpp)** &nbsp;`C++`  
Soluções estruturadas para problemas de algoritmo, organizadas por tópico e dificuldade. Comentários explicam o raciocínio, não só o código.

**[habit-tracker](https://github.com/KauePelosi/habit-tracker)** &nbsp;`TypeScript · HTML · CSS`  
App web leve para monitoramento de hábitos. Sem backend, sem contas — só localStorage e UX focada. Feito como ponto de entrada prático em TypeScript.

**Em andamento:** &nbsp; Servidor HTTP em C++ &nbsp;·&nbsp; Projetos com Angular &nbsp;·&nbsp; Ferramentas em Python
